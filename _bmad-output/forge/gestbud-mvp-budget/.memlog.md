- (decision) Cible principale : utilisateur en Afrique francophone (Sénégal, CI, Cameroun, etc.) — marché où les APIs bancaires ouvertes n'existent pas, transactions cash dominantes, pas d'équivalent local à Bankin/Linxo. L'OCR de reçus physiques devient la feature centrale, pas un bonus.
- (decision) Produit cible : OCR (reçus physiques) + intégration Mobile Money (Wave, Orange Money, MTN MoMo). Les deux dans la vision produit.
- (lock) MVP = OCR (reçus physiques formels) + auth + saisie manuelle + catégories + calcul mensuel + visualisation solde. Mobile Money repoussé en V2 (nécessite partenariats commerciaux).
- (lock) Utilisateur cible MVP : urbain (Dakar, Abidjan, Douala) avec 4G fiable. OCR cloud validé (Google Vision / Mindee). Pas de contrainte offline pour le MVP.
- (lock) Monétisation MVP : gratuit. Objectif = prouver l'usage. Revenus et conversion freemium/payant repoussés en V2 avec Mobile Money.
- (lock) Plateforme MVP : PWA mobile-first. Android dominant sur le marché cible. Pas de publication sur stores nécessaire pour le MVP. Accès caméra via navigateur pour l'OCR.
- (lock) Auth MVP : numéro de téléphone + OTP SMS via Africa's Talking. Email écarté (usage marginal sur le marché cible). Coût marginal ~0,02€/SMS acceptable pour un MVP gratuit.
