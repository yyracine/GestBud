- (constraint) PRD final (2026-06-30) : Flutter, SQLite local (sqflite), Mindee OCR + Google Vision fallback, Africa's Talking OTP, XOF V1 / XAF V2, iOS 15+ / Android 9+, interface française, stockage 100% local sans backend
- (decision) Paradigme d'état : Riverpod (flutter_riverpod). Binds: tout provider d'état dans l'app utilise Riverpod. Prevents: mélange BLoC/Provider/setState pour la logique métier. Rule: le Solde et les agrégats du tableau de bord sont des providers dérivés (pas recalculés côté UI).
- (decision) Catégorisation IA : appel LLM batch (Claude Haiku) après retour OCR Mindee, toutes les lignes en une seule requête. Binds: la catégorisation passe par CategorizationService (interface), pas directement dans le provider de scan. Prevents: appels LLM ligne par ligne (latence), logique de catégorisation couplée au flux scan. Rule: dictionnaire de mots-clés minimal comme fallback si l'appel LLM échoue/timeout.
- (decision) Structure du code : feature-based. lib/features/{auth,scan,transactions,dashboard,categories}/ chacune autonome. lib/shared/ pour les modèles Transaction/Catégorie, le schéma SQLite, et les providers transversaux (solde). Binds: chaque feature encapsule ses propres providers, widgets, et accès data. Prevents: couplage direct entre features (une feature passe par shared/ pour lire les données d'une autre).
- (decision) Base de données locale : Drift (ORM type-safe au-dessus de sqflite). Binds: tout accès SQLite passe par des DAOs Drift générés. Prevents: requêtes SQL en string brut, migrations manuelles. Rule: le schéma inclut un champ currency sur User/Settings dès V1 pour accommoder XAF en V2 sans migration cassante.
- (decision) Navigation : GoRouter (go_router, maintenu Flutter team). Binds: toutes les routes de l'app sont nommées et déclarées dans un GoRouter central. Prevents: navigation impérative dispersée (Navigator.push) pour les routes principales.
- (decision) Proxy API (BFF) : Cloudflare Worker stateless. L'app Flutter appelle uniquement le BFF ; le BFF appelle Mindee (OCR), Mistral (catégorisation batch), Africa's Talking (OTP). Binds: aucune clé API tierce n'est embarquée dans le binaire Flutter. Prevents: exposition des clés dans l'APK/IPA. Rule: le BFF est stateless — aucune donnée utilisateur n'y transite ni n'y est stockée. LLM : Mistral (fort support français, pertinent pour les libellés de reçus africains).
- (decision) Persistance session : flutter_secure_storage (keychain iOS / Keystore Android). Binds: le token de session est exclusivement stocké dans flutter_secure_storage. Prevents: stockage du token dans SharedPreferences ou base Drift (non chiffré). Rule: correction du PRD qui mentionne localStorage (concept web, inapplicable mobile).
- (decision) Schéma : table transactions avec receipt_id nullable (String?). Transactions manuelles : receipt_id = null. Lignes d'un même reçu : même receipt_id (UUID généré au moment du scan). Binds: le groupement visuel dans l'historique se base sur receipt_id, pas sur une logique applicative. Prevents: deux implémentations incompatibles du groupement reçu.
